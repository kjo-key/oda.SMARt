Information and Computer Security

Key Concepts in Information and Computer Security

Confidentiality: Ensures that information is only accessible to authorized individuals or systems.

Integrity: Ensures that the data is accurate, consistent, and has not been tampered with.

Availability: Ensures that data and services are accessible when needed by authorized users.

Types of Security Threats

  • Malware: Software designed to harm or exploit any device or network. Examples include viruses, worms, and ransomware.
  • Phishing: A method where attackers deceive users into revealing sensitive information like passwords or credit card details.
  • Denial of Service (DoS) / Distributed Denial of Service (DDoS): Attacks aimed at making a network service unavailable by overwhelming it with traffic.
  • SQL Injection: An attack that involves inserting malicious SQL queries into input fields to manipulate a database.
  • Man-in-the-Middle (MitM): An attack where an attacker intercepts and potentially alters communication between two parties.
  • Social Engineering: The use of manipulation to trick people into divulging confidential information.

Cryptography and Encryption

1. Symmetric Encryption

Both the sender and receiver use the same key for encryption and decryption. Examples: AES, DES.


C = E_k(P)  // Encryption
P = D_k(C)  // Decryption

2. Asymmetric Encryption

Uses a pair of keys: a public key for encryption and a private key for decryption. Examples: RSA, ECC.


C = E_Public(P)  // Public Key Encryption
P = D_Private(C)  // Private Key Decryption

3. Hash Functions

A one-way function that converts input data into a fixed-size string of characters. Examples: MD5, SHA-256.


H = hash(P)  // Hashing function

Authentication and Access Control

Authentication: Verifying the identity of a user, device, or system.

Access Control Models:

  • Discretionary Access Control (DAC): The owner of the resource determines who can access it.
  • Mandatory Access Control (MAC): The system dictates who can access resources based on policies.
  • Role-Based Access Control (RBAC): Access is based on the roles assigned to users.

Security in Networks

  • Firewalls: Security systems that monitor and control incoming and outgoing network traffic.
  • Intrusion Detection Systems (IDS): Systems that detect unauthorized access attempts.
  • Virtual Private Networks (VPN): A secure network that allows remote access by encrypting data transferred over the internet.
  • SSL/TLS: Protocols for encrypting data between web browsers and servers.

Security in Software Development

  • Secure Coding Practices: Writing software that is resistant to vulnerabilities.
  • Code Audits: Reviewing code to identify and mitigate security vulnerabilities.
  • Patch Management: Keeping software up to date with security patches.
